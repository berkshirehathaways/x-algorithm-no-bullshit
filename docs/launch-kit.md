## Reddit

**Title:** I made a source-checked guide to what X’s public algorithm code actually tells us

**Body:**

A lot of discussion about X ranking combines useful but older material with newer architecture. I wanted a clearer way to say what the public source confirms, what is legacy-only, and what remains unknown.

X Algorithm: No Bullshit is an independent, provenance-first repository. It tracks machine-readable claims with a verdict and citation. The current public `RankingScorer` combines 22 named score terms and obtains their weights through runtime feature-switch parameters. The snapshot does **not** establish the live service or current numeric values, so the project does not present a “2026 weight list.”

The aim is not to promise better reach or virality. It is to make explanations easier to audit. Thoughtful corrections with primary sources are welcome.

## 72-hour distribution checklist

### Before launch

- [ ] Confirm every post uses the canonical repository URL.
- [ ] Open each cited upstream URL and confirm the cited commit is still reachable.
- [ ] Read every launch post against the claim registry; remove unsupported phrasing.
- [ ] Publish the README and claim sources before sharing any post.
- [ ] Prepare a short disclosure: independent project; not affiliated with X.

### 0–24 hours

- [ ] Publish the launch post and one thread from the project account or a clearly identified personal account.
- [ ] Submit the Hacker News post once; participate candidly in replies.
- [ ] Post to the most relevant Reddit community only when its rules permit it; disclose authorship.
- [ ] Respond to factual corrections with citations or amend the repository.
- [ ] Record recurring questions as candidate claims, not as assumptions.

### 24–48 hours

- [ ] Share one concrete claim card that links to its current and legacy sources.
- [ ] Ask domain peers for source review, not likes, reposts, or coordinated comments.
- [ ] Fix attribution, wording, and broken links promptly; note substantive corrections publicly.
- [ ] Add only evidence-backed answers to discussion threads.

### 48–72 hours

- [ ] Publish a brief changelog of verified corrections and newly sourced claims.
- [ ] Review referral and discussion quality without treating attention as validation.
- [ ] Archive feedback that identifies missing provenance or unclear verdicts.
- [ ] Stop channels that reject self-promotion or where the project is off-topic.
